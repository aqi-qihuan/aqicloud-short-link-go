## 性能分析与压测方案

对 aqicloud-short-link-go 项目进行了全面的性能分析，识别出 9 个性能瓶颈：

### P0 (致命级)
1. **热路径无 Redis 缓存** — `link_api.go:50` 每次短链重定向都查 MySQL，这是性能差的根本原因
2. **Gateway 每次请求新建 Proxy** — `gateway/main.go:42-51` 每次都 url.Parse + NewSingleHostReverseProxy，GC 压力巨大

### P1 (严重级)
3. **GORM 未配置连接池** — `link/main.go:42` 无 MaxOpenConns 设置
4. **限流器内存泄漏** — `ratelimit.go:50` sync.Map 无 TTL 清理
5. **RabbitMQ 单 Channel** — `rabbitmq.go:17` 整个应用只有 1 个 Channel
6. **Kafka BatchTimeout 值错误** — `kafka.go:22` `BatchTimeout: 10` 实际是 10ns 而非 10ms

### P2 (中等级)
7. MQ 消费者单协程
8. MQ 发布未确认 (Publisher Confirms)
9. Gateway 使用 gin.Default() 多余日志
